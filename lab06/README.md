# Студенческий портал — Лабораторные работы 6.1 и 6.2

Многостраничное React-приложение, использующее **react-router-dom** для маршрутизации на стороне клиента, с динамическими маршрутами и параметрами запроса.

## Структура маршрутов

```
/ (Макет)
├── / (индекс) → Главная
├── /courses → Курсы (список с ?sort=asc|desc)
├── /courses/:id → Подробная информация о курсе (динамическая)
├── /about → О нас
└── * → NotFound (404)
```

## useParams против useSearchParams

| Хук | Назначение | Когда использовать |

|------|---------|-------------|

| **useParams** | Возвращает динамические сегменты URL (например, `:id` в `/courses/:id`) | Когда параметр идентифицирует конкретный ресурс. Например, `/courses/1` — `1` определяет, какой курс отображать. |

| **useSearchParams** | Возвращает строку запроса в виде `URLSearchParams` (например, `?sort=asc`) | Когда параметр необязателен, фильтрует/сортирует данные или не изменяет идентификатор страницы. Например, `?sort=asc` — та же страница, другой порядок. |

**Параметры URL** (`/courses/1`): Используется для идентификаторов ресурсов — каждое значение указывает на отдельную сущность. Обязателен для корректной работы маршрута.

**Параметры запроса** (`?sort=asc`): Используйте для необязательного состояния — фильтры, сортировка, пагинация. Страница работает и без них; они просто изменяют поведение.

## Возможности лабораторной работы 6.2

- **Динамический маршрут** `/courses/:id` — подробная информация о курсе с помощью `useParams` и `useLoaderData`
- **Загрузчик** — получает данные о курсе перед рендерингом; выдает исключение при недопустимых ID
- **Элемент ошибки** — удобное для пользователя сообщение «Курс не найден» для недопустимых ID (например, `/courses/999`)
- **Сортировка** — `?sort=asc` / `?sort=desc` с помощью `useSearchParams`; кнопка сортировки переключает порядок URL и списка
